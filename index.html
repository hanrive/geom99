PROBLEM

Ideal

In an ideal scenario, collecting spatial data from the public and mapping live and public data sources would be seamless, ensuring accurate, real-time information that enhances urban planning, environmental monitoring, and community engagement.

Reality

However, in reality, this process is fraught with challenges such as privacy concerns, data accuracy issues, and the technological divide, which hinder the efficient collection and the use of spatial data.

Consequences

These challenges lead to underutilized spatial data, missed opportunities for data-driven decision-making in critical areas like disaster response and urban development, and increased skepticism among the public about data privacy and usage.

Proposal

To address these issues, it's proposed to implement a robust web-based framework that prioritizes data privacy, enhances data accuracy through advanced validation techniques, and promotes digital inclusivity, thereby improving the collection and mapping of spatial data for the benefit of all stakeholders.

Technology Used

ArcGIS Dashboard, ArcGIS Hub, ArcGIS Survey123, KoboToolbox.

Clappia

Link to created survey: https://geom99.clappia.com/app/INC163980

Clappia is a no-code platform that allows users to build applications with extensive customization capabilities. Designing apps is amazingly simple with a drag and drop interface, and all stored within a centralized workspace. Clappia integrates with various data sources and offers features like workflow automation, form building, and data visualization. Its user-friendly design empowers businesses to streamline processes and improve productivity without relying on technical expertise. Here is the link to the website: https://www.clappia.com/ It has four tiers of payment. The plan we used to test out the site was the free plan which offers: up to 100 users, 400 submissions per month, 400 workflow step executions per month, unlimited apps, 1 PDF configuration, 1 Google Sheet Integration, 20 free messages per month over both WhatsApp and SMS, and support over email and chat.

You can use an already made template with themes of daily attendance tracker, restaurant order taking solution, inventory management, IT ticketing solution, and much more. This app for incident reporting was made from scratch and followed the same format with the same questions as the survey made in Survey123.

This gif shows the process of data collection on the survey on a phone; however, users can also use a computer to submit data. It generates a unique ticket per submission automatically, collects spatial data using Google Maps, collects the time and data, and more. In the workspace dashboard (the home page of the app) there is a tab for submissions (under â€˜Submissionsâ€™) where you can see the all the responses made, as well as the answers/owner of submission/etc., as well as a tab to see the analytics (under â€˜Analyticsâ€™) where you can display the location of all submissions as well as capabilities to display answers with a chart (I couldnâ€™t figure out how to do that exactly though).

There is also the option to have the data automatically send the results to a Google Sheets/Drive or integrate a database. When looking at database integration, it gives the options for database type of MySql, PostgreSql, and AzureSql. We tried connecting the Clappia survey to ArcGIS Online with each type of database however the following problems were encountered: MySql only allowed the option of downloading and manually publishing the data to Online through ArcGIS Pro or the implementation of ArcGIS Server or a full ArcGIS Enterprise deployment to serve the data as a REST service. PostgreSql and AzureSql both suggested if we were to have ArcGIS Enterprise (which we do not) we could register a PostgreSQL database with ArcGIS Server and then publish the map/feature to ArcGIS Server and then to ArcGIS Online. Overall, no automated method that allowed pushing to ArcGIS without manual intervention or a more expensive license.

Takeaways:

  • Allowed both organizations and the public to submit data (with the use of a link)
  • Allows full customization of applications with workflows, reports, dashboards, and integrations
  • Hard to integrate with other platforms, a lot of limitations with the free pay tier, not built with geospatial data collection and integration with GIS systems in mind

Survey 123

Link to created survey: Pollution Reporting(survey123.arcgis.com)

Survey123 is an esri product form-centric solution for creating and sharing surveys, getting answers, and analyzing results. Use it to create forms with skip logic, defaults, and support for multiple languages. Collect data using web or mobile devices, even when disconnected from the internet. Upload data securely, and analyze results on the web or in an ArcGIS app. Link to the website: https://survey123.arcgis.com// All Survey123 items (forms, web maps, tabular reference data, and survey records) are stored in your ArcGIS Online or ArcGIS Enterprise organization. The Survey123 website is used to manage all surveys, view and analyze data, and create feature reports. When authoring forms and collecting records, you can use the Survey123 component that suits your needs. You can author surveys in Survey123 Connect or the web designer, and complete surveys in the field app or web app.

To create a survey form you need start creating a new survey or an old survey created by you can be edited to make it relatable to the current topic of yours upload an XLSForm, or important an XLSForm via URL. These survey forms can contain text, audio, or images as explanatory information for the survey respondent and allow the input of the same. Surveys can be as free-form or prescriptive as the author chooses. Questions can be hidden or visible based on previous choices, selections can be limited based on external data, and JavaScript functions can be used to automate form behavior.

The video shows the structure of the survey created using Survey123 and view of the survey which is opened on web browser by the user to collect data. The survey can be opened using any web browser and the survey can be filled. The Survey is kept public which enables anyone to fill out the survey and report the incident.

Benefits of Survey123:

  • It is easy to create survey forms using Survey123 and anyone with basic information of GIS can create the survey.
  • Survey 123 needs minimal training is enough for anyone who wants to collect data.
  • Survey123 can collect data in form of location, photographs, date and time, text and numeric values.
  • Data collected using survey 123 can be integrated with ArcGIS platforms such as Dashboard.
